Federal Data Strategy Compliance

How this catalog satisfies six federal data governance requirements.

Compliant
OMB M-19-18

"Agencies must maintain a comprehensive data inventory."

331 tables across 8 schemas with column-level metadata, grain documentation, and foreign key relationships cataloged — 8,241 columns in total.

Compliant
OPEN Government Data Act

"Publish machine-readable data catalog."

Structured YAML definitions compiled to a searchable JSON index. All schema definitions are version-controlled and programmatically accessible.

Compliant
OMB Circular A-130

"Manage information as a strategic resource."

Metadata routing, grain documentation, PII classification, and access controls applied at the column level across all 8 operational schemas.

Compliant
NIST 800-53 CM-8

"Maintain a component inventory."

Schemas, tables, columns, foreign keys, and ERDs cataloged with full provenance — source system, extraction method, and last-validated timestamp.

Compliant
FISMA / FedRAMP Classification

"Classify data by sensitivity level."

Three-tier PII classification (None / Limited / Full) with Row-Level Security enforcement. All Tier 2 and Tier 3 tables require elevated role assignment.

Compliant
OMB M-23-22 Data Governance

"Designate data stewards and access policies."

Role-based access maps stewards, analysts, and admins to schemas and PII tiers. Steward assignments documented per schema with quarterly review cadence.

Join Pattern

Charge to District Join

Standard join from ChargeInquiryTransform1 to the District reference table using the accountability district code.

SELECT c.*, d.DistrictName
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.District d
  ON c.AccountabilityDistrictCode = d.DistrictCode
Join Pattern

Charge to Office Join

Joins charges to the receiving office for intake and routing queries. Use ReceivingOfficeCode for intake location.

SELECT c.*, o.OfficeName, o.OfficeType
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.Office o
  ON c.ReceivingOfficeCode = o.OfficeCode
Join Pattern

ADR Charge Benefits Join

Links charge records to ADR benefit outcomes. Use this pattern for settlement amount and non-monetary relief queries.

SELECT c.ChargeNumber, b.BenefitType,
       b.MonetaryAmount, b.NonMonetaryCode
FROM accessory.ChargeInquiryTransform1 c
JOIN adr.charge_benefit b
  ON c.ChargeNumber = b.charge_number
WHERE b.is_active = 1
Join Pattern

Case Folder to Resolution

Joins case folder to final resolution outcome. Required for all close basis and resolution date queries.

SELECT cf.CaseFolderID, cf.OpenDate,
       cr.CloseBasis, cr.ResolutionDate,
       cr.MonetaryRelief
FROM private.case_folder cf
JOIN private.charge_resolution cr
  ON cf.CaseFolderID = cr.case_folder_id
Business Rule

Fiscal Year Boundary

EEOC fiscal year runs October 1 through September 30. All FY-based queries must use this boundary definition.

-- FY boundary rule
-- FY start: October 1  (month 10, day 1)
-- FY end:   September 30  (month 9, day 30)
-- Example: FY2025 = 2024-10-01 to 2025-09-30
WHERE IntakeDate >= '2024-10-01'
  AND IntakeDate <  '2025-10-01'
Business Rule

Open Charge Definition

A charge is considered open when its close date is NULL and close basis code is not set. Use this filter to avoid counting resolved charges in active caseload queries.

-- Open charge filter rule
WHERE c.CloseDate IS NULL
  AND c.CloseBasisCode IS NULL
  AND c.ChargeStatusCode NOT IN ('CL', 'WD', 'DS')
-- 'CL' = closed, 'WD' = withdrawn, 'DS' = dismissed
Schema Rule

Historical Data via retro Schema

Queries spanning pre-migration periods must join retro schema tables. The retro schema mirrors private schema structure with an added historical period column.

-- Use retro schema for pre-migration charge data
-- retro.charge_inquiry covers charges prior to IMS cutover
-- Join key: charge_number (composite with period_id)
SELECT * FROM retro.charge_inquiry
WHERE period_id = 'LEGACY'
  AND charge_number = :charge_number

Example natural language queries with tables used and SQL
Question Tables Used SQL
How many charges were filed last fiscal year? ChargeInquiryTransform1 
SELECT COUNT(*) AS charge_count
FROM accessory.ChargeInquiryTransform1
WHERE IntakeDate >= '2024-10-01'
  AND IntakeDate <  '2025-10-01';
How many inquiries are currently open? ChargeInquiryTransform1 
SELECT COUNT(*) AS open_inquiries
FROM accessory.ChargeInquiryTransform1
WHERE CloseDate IS NULL
  AND CloseBasisCode IS NULL
  AND ChargeStatusCode NOT IN ('CL', 'WD', 'DS');
What is the accountability district for each charge? ChargeInquiryTransform1, District 
SELECT c.ChargeNumber, d.DistrictName AS AccountabilityDistrict
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.District d
  ON c.AccountabilityDistrictCode = d.DistrictCode
ORDER BY c.ChargeNumber;
What is the receiving district for each charge? ChargeInquiryTransform1, District 
SELECT c.ChargeNumber, d.DistrictName AS ReceivingDistrict
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.District d
  ON c.ReceivingDistrictCode = d.DistrictCode
ORDER BY c.ChargeNumber;
When was the intake interview held for each charge? ChargeInquiryTransform1, charge_event_log 
SELECT c.ChargeNumber, e.EventDate AS IntakeInterviewDate
FROM accessory.ChargeInquiryTransform1 c
JOIN private.charge_event_log e
  ON c.ChargeNumber = e.charge_number
WHERE e.EventTypeCode = 'INTAKE_INTERVIEW'
ORDER BY c.ChargeNumber;
What is the settlement rate by district this fiscal year? ChargeInquiryTransform1, District, charge_resolution 
SELECT d.DistrictName,
       COUNT(*) AS total_closed,
       SUM(CASE WHEN cr.CloseBasisCode = 'SET' THEN 1 ELSE 0 END) AS settled,
       ROUND(100.0 * SUM(CASE WHEN cr.CloseBasisCode = 'SET' THEN 1 ELSE 0 END)
             / COUNT(*), 1) AS settlement_rate_pct
FROM private.charge_resolution cr
JOIN private.case_folder cf ON cr.case_folder_id = cf.CaseFolderID
JOIN accessory.ChargeInquiryTransform1 c ON cf.ChargeNumber = c.ChargeNumber
JOIN accessory.District d ON c.AccountabilityDistrictCode = d.DistrictCode
WHERE cr.ResolutionDate >= '2025-10-01'
  AND cr.ResolutionDate <  '2026-10-01'
GROUP BY d.DistrictName
ORDER BY settlement_rate_pct DESC;
How many ADR mediations reached settlement last quarter? mediation_session, charge_benefit 
SELECT COUNT(DISTINCT ms.charge_benefit_id) AS settled_mediations
FROM adr.mediation_session ms
JOIN adr.charge_benefit cb ON ms.charge_benefit_id = cb.id
WHERE ms.SessionOutcomeCode = 'SETTLED'
  AND ms.SessionDate >= DATEADD(quarter, -1, GETDATE());
List all charges by basis code for a given office. ChargeInquiryTransform1, Office 
SELECT c.ChargeNumber, c.BasisCode, c.BasisDescription, o.OfficeName
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.Office o ON c.ReceivingOfficeCode = o.OfficeCode
WHERE o.OfficeCode = :office_code   -- bind param
ORDER BY c.IntakeDate DESC;
What is the average days-to-close by resolution type? case_folder, charge_resolution 
SELECT cr.CloseBasisCode,
       AVG(DATEDIFF(day, cf.OpenDate, cr.ResolutionDate)) AS avg_days_to_close,
       COUNT(*) AS case_count
FROM private.case_folder cf
JOIN private.charge_resolution cr ON cf.CaseFolderID = cr.case_folder_id
WHERE cr.ResolutionDate IS NOT NULL
GROUP BY cr.CloseBasisCode
ORDER BY avg_days_to_close;
How many systemic investigations are currently active? systemic_case_charge, case_folder 
SELECT COUNT(DISTINCT scc.systemic_case_number) AS active_systemic
FROM private.systemic_case_charge scc
JOIN private.case_folder cf ON scc.case_folder_id = cf.CaseFolderID
WHERE cf.CloseDate IS NULL
  AND scc.is_active = 1;
What roles have access to the private schema? access_role, access_domain 
SELECT ar.RoleName, ar.PermissionLevel,
       ar.PIITierAccess, ad.DomainName
FROM admin.access_role ar
JOIN admin.access_domain ad ON ar.domain_id = ad.id
WHERE ar.SchemaGrants LIKE '%private%'
ORDER BY ar.PermissionLevel DESC;
List total monetary relief awarded by fiscal year. charge_resolution, ChargeInquiryTransform1 
SELECT YEAR(cr.ResolutionDate) AS fiscal_year_end,
       SUM(cr.MonetaryRelief) AS total_monetary_relief,
       COUNT(*) AS resolutions_with_relief
FROM private.charge_resolution cr
WHERE cr.MonetaryRelief > 0
GROUP BY YEAR(cr.ResolutionDate)
ORDER BY fiscal_year_end DESC;
Which FEPA offices filed the most charges last year? ChargeInquiryTransform1, Office 
SELECT o.OfficeName, o.OfficeType, COUNT(*) AS charges_filed
FROM accessory.ChargeInquiryTransform1 c
JOIN accessory.Office o ON c.ReceivingOfficeCode = o.OfficeCode
WHERE o.OfficeType = 'FEPA'
  AND c.IntakeDate >= '2024-10-01'
  AND c.IntakeDate <  '2025-10-01'
GROUP BY o.OfficeName, o.OfficeType
ORDER BY charges_filed DESC;
What is the case load per mediator by district? mediator_roster, mediation_session, District 
SELECT d.DistrictName,
       COUNT(DISTINCT mr.MediatorID) AS mediator_count,
       COUNT(DISTINCT ms.charge_benefit_id) AS active_cases,
       ROUND(1.0 * COUNT(DISTINCT ms.charge_benefit_id)
             / NULLIF(COUNT(DISTINCT mr.MediatorID), 0), 1) AS cases_per_mediator
FROM adr.mediator_roster mr
JOIN adr.mediation_session ms ON mr.MediatorID = ms.MediatorID
JOIN accessory.District d ON mr.DistrictCode = d.DistrictCode
WHERE mr.is_active = 1
  AND ms.SessionOutcomeCode = 'OPEN'
GROUP BY d.DistrictName
ORDER BY cases_per_mediator DESC;
