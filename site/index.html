Integration

End-to-End Charge Lifecycle

From the moment a charge is filed in ARC through final disposition and analytics — every step connected.

1

Charge Filed in ARC

A discrimination charge is filed at an EEOC field office and entered into the Agency Record Center (ARC), the legacy system of record.

ARC stores charge data, investigation records, and enforcement history for all Title VII, ADA, ADEA, EPA, and GINA filings.
2

ARC Integration API Stages the Case

The Integration API pulls eligible cases from ARC every 15 minutes via authenticated REST calls, translating legacy SOAP/REST interfaces into clean JSON endpoints.

GET /arc/v1/mediation/eligible-cases → watermark-based pagination, SSRF protection, managed identity auth
3

Triage Routes to the Right Track

AI classification analyzes the charge narrative, identifies applicable statutes, and recommends the optimal resolution path: mediation, investigation, or conciliation.

GPT-4o with stop sequences ["Legal Advice:", "Legal Conclusion:"] → human review required before any action
4

ADR Manages Mediation

The mediator picks up the case, schedules sessions, manages documents, and uses AI to draft settlement agreements — all with HMAC-signed audit trails and 7-year WORM retention.

HMAC-SHA256 signed audit records → Azure Blob immutable storage → NARA-compliant 7-year retention
5

OGC Prepares for Trial (if needed)

If mediation fails and the case proceeds to litigation, attorneys use the Trial Tool for document analysis, timeline construction, and AI-assisted legal research across the full case file.

RAG-powered document search → Azure AI Search + GPT-4o vision for scanned documents
6

Analytics Dashboard Aggregates Outcomes

Metrics from every stage flow into UDAP. Leadership sees settlement rates, time-to-resolution, office performance, and charge trends — queryable in natural language.

Daily metrics push → data-middleware ETL → PostgreSQL → Superset dashboards + AI assistant chat

Security Posture

Defense in Depth

Every commit runs through 15+ security gates before reaching production. Six advisory databases, static and dynamic testing, supply chain verification, and tamper-evident audit trails.

 6-Database SCA

OWASP Dependency-Check, pip-audit, Trivy, Dependabot, OSV-Scanner, and Grype scan every dependency against NVD, PyPI, GitHub Advisory, Google OSV, and Anchore feeds.

 SAST + PII Rules

Bandit and Semgrep with custom PII leak detection rules. Four rule categories catch PII in log statements, API responses, exception messages, and audit records.

 Authenticated DAST

ZAP active scan with valid credentials tests all authenticated endpoints for injection, XSS, auth bypass, and OWASP Top 10 vulnerabilities.

 Secret Scanning

Gitleaks scans the full git history on every commit. Pre-commit hooks prevent secrets from being committed. CI gates fail the build on any finding.

 HMAC Audit Trail

Every AI generation is signed with HMAC-SHA256 and stored in immutable Azure Blob storage with 7-year WORM retention. NARA-compliant — verified during 2025 OIG inquiry.

 Supply Chain

CycloneDX SBOM generation, Syft system dependency scanning, ScanCode snippet-level license detection, and SLSA + Cosign container image signing with build provenance.
