Model Context Protocol

MCP Hub — Protocol and Tool Catalog

The Hub implements MCP over streamable HTTP. Each spoke registers its tools at startup. The Hub merges all spoke catalogs into a single endpoint for AI agent consumers.

JSON-RPC 2.0 over Streamable HTTP

All MCP calls use JSON-RPC 2.0 request/response framing transported over HTTPS. Streaming responses (for long-running tool calls) use HTTP chunked transfer. No WebSocket dependency — compatible with APIM throttling and request logging.

Tool Discovery — 5-Minute Cycle

The Hub polls each spoke's /api/mcp/tools health endpoint every 5 minutes. New tools registered by a spoke app appear in the merged catalog within one poll cycle. Spokes that fail health checks are marked unavailable and removed from the catalog until they recover.

Spoke Registration Contract

Each spoke registers four fields: name (unique identifier), url (base URL for tool calls), auth_scope (the Entra ID scope the Hub requests), and health_endpoint (polled every 5 minutes). Authentication uses Managed Identity — no shared secrets.

Sample Tool Catalog Response

GET /api/tools returns the merged catalog from all active spokes. Each entry includes the tool schema so AI agents can construct calls without additional lookups.

1

ARC to Integration API

The Integration API polls ARC every 15 minutes using watermark-based pagination. Only cases eligible for mediation are retrieved. The caller authenticates with Managed Identity; SSRF protections prevent requests to unintended hosts.

GET /arc/v1/mediation/eligible-cases?watermark={ts}&limit=100 → 200 OK (JSON)
2

Staging in Azure Table Storage

Retrieved charge records land in Azure Table Storage with a pending row status. The Integration API deduplicates by charge number before writing. No PII is logged at any point in this path.

Table: arc_charges_staging — PartitionKey: office_id, RowKey: charge_number
3

ADR Picks Up Pending Cases

An ADR Azure Function triggers on the Table Storage queue every 5 minutes, reads pending rows, creates case records in PostgreSQL, assigns them to the appropriate district office queue, and marks the staging row processed.

AzureStorageQueueTrigger → create_case() → UPDATE staging SET status='processed'
4

ADR Pushes Daily Metrics to UDAP

Each night, an ADR Function App publishes aggregated outcome metrics (settlement rate, time-to-resolution, disposition counts by office) to the UDAP data-middleware ingest endpoint. No individual charge data or PII crosses this boundary — only aggregates.

POST /ingest/adr/daily-metrics → data-middleware ETL → PostgreSQL → Superset
Authentication

Authentication Architecture

Four distinct authentication paths cover every actor in the platform. All paths terminate at Azure Entra ID. No application stores credentials — secrets live in Key Vault, service-to-service auth uses Managed Identity.

EEOC Staff
Azure Entra ID (OIDC)

Staff authenticate with their @eeoc.gov identity via Azure Entra ID. Browser-based applications use the standard OIDC authorization code flow. Sessions are stored in Redis — no cookie serialization.

Entra ID → OIDC auth code → Redis session
Scope: openid profile email groups
External Parties
Login.gov OIDC + PKCE

Respondents, charging parties, and other external users authenticate through Login.gov. The platform uses PKCE with private_key_jwt client assertion — no client secret transmitted over the network.

Login.gov → PKCE + private_key_jwt → Redis session
IAL2 identity assurance required for document submission
Service-to-Service
Managed Identity

Inter-service calls (Hub to spokes, Function Apps to storage, Integration API to ARC) use Azure Managed Identity. No credentials are configured, rotated, or stored — identity comes from the Azure control plane.

Managed Identity → Entra ID token → target resource
X-Request-ID propagated through every hop (RFC 7807)
AI Audit Trail
HMAC-SHA256 Signed Records

Every AI generation — prompt, response, model version, and timestamp — is signed with HMAC-SHA256 using a key stored in Key Vault. Records are written to Azure Blob immutable storage with a 7-year WORM lock. NARA-compliant; verified during the 2025 OIG inquiry.

AI output → HMAC-SHA256(KV key) → Blob WORM
Retention: 7 years per NARA 2023-0002
