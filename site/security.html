Internal Advantages

Tools Neither Black Duck nor Seeker Provide

Several categories are outside the scope of both commercial products entirely. These are enforced in every repo through the local CI sweep.

 Gitleaks — Secret Scanning

Scans the full git commit history on every push. Pre-commit hooks prevent credentials from reaching the remote. CI gates fail the build on any finding.

 Checkov — IaC Scanning

Scans Kubernetes manifests, Dockerfiles, and Bicep/Terraform for misconfigurations — exposed ports, overprivileged service accounts, missing network policies.

 SLSA + Cosign — Supply Chain Provenance

Container images are signed at build time via Cosign keyless OIDC. SLSA provenance attestations link every image back to the source commit and build pipeline.

 Hypothesis — Property-Based Fuzzing

Generates thousands of inputs from type contracts and verifies invariants hold under adversarial conditions. Catches classes of bugs that fixed test cases miss — particularly boundary and encoding issues.

 HMAC Audit Integrity (AU-10)

Every AI generation is signed with HMAC-SHA256 before storage. Signatures are verified on retrieval. Non-repudiation is enforced at the application layer, not just at the infrastructure layer.

 508 Accessibility Scanning

axe-core accessibility scanning (vendored library for template testing) runs alongside a custom Semgrep rule set that catches missing ARIA attributes and contrast failures at the source level. Section 508 is federal law — Semgrep findings block the build.

 7-Year WORM Retention (NARA)

Audit records are written to Azure Blob immutable storage with a 7-year retention lock. Required by NARA for AI-generated federal records. Verified during a 2025 OIG inquiry that required retrieval of 18-month-old records.

Assessment

Where the Gaps Are

Black Duck queries two vulnerability databases (NVD and its proprietary BDSA feed). The internal toolset queries six. Black Duck's one concrete advantage is the BDSA advisory library — approximately 2,000 entries that NVD does not yet carry, with a 2 to 14 day early-warning window before the NVD record is published. On license detection, the two stacks reach parity through ScanCode's snippet matching. The only Black Duck capability without an internal equivalent is its legal workflow dashboard for tracking license approvals across a legal team.

Seeker instruments the running application from inside the process. It can trace a specific HTTP input through five internal function calls and confirm it reaches a vulnerable sink — at runtime, against real execution paths. The internal toolset approaches this from two directions: ZAP runs authenticated DAST from outside the HTTP boundary, and four Semgrep rules block PII patterns at build time before the code ships. Property-based fuzzing with Hypothesis validates function-level invariants. None of these replicate the inside-out tracing that IAST provides. That is the only architectural gap in the internal security stack.
